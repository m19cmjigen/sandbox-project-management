# プロジェクト紐付け

プロジェクト紐付けでは、組織に割り当てられていない「未分類プロジェクト」を組織に紐付ける操作を行います。

## 画面へのアクセス

ナビゲーションバーの **「管理」** → **「組織管理」** をクリックします。

ページ右側の **「未分類プロジェクト」** パネルから操作します。

## 未分類プロジェクトパネル

```
┌──────────────────────────────────────────┐
│ 未分類プロジェクト  [3]                  │
│──────────────────────────────────────────│
│ PROJ-X  プロジェクトX          [割当アイコン] │
│         lead@example.com                 │
│──────────────────────────────────────────│
│ PROJ-Y  プロジェクトY          [割当アイコン] │
│──────────────────────────────────────────│
│ PROJ-Z  プロジェクトZ          [割当アイコン] │
└──────────────────────────────────────────┘
```

- 件数バッジ（例: `[3]`）に未分類プロジェクトの数が表示されます
- 未分類プロジェクトがない場合は「未分類のプロジェクトはありません」と表示されます

## プロジェクトを組織に割り当てる

1. 「未分類プロジェクト」パネルで割り当てたいプロジェクトの **「割当アイコン」** をクリックする

2. 「組織に紐付け」ダイアログが表示される

```
┌────────────────────────────────────┐
│ 組織に紐付け                       │
│                                    │
│ プロジェクト: PROJ-X プロジェクトX  │
│                                    │
│ 紐付け先組織: [組織を選択してください ▼] │
│                                    │
│       [キャンセル]  [紐付け]        │
└────────────────────────────────────┘
```

3. **「紐付け先組織」** ドロップダウンから割り当て先の組織を選択する

   - ドロップダウンには全組織が階層レベルのインデント付きで表示されます
   - 任意のレベルの組織（本部・部・課）を選択できます

4. **「紐付け」** ボタンをクリックする

5. 成功すると「○○を組織に紐付けました」というメッセージが表示され、プロジェクトが未分類パネルから消えます

## 割り当てを解除する

現在の実装では、UIから直接割り当てを解除することはできません。

割り当てを解除するには、同じプロジェクトを再度別の組織に紐付けることで変更できます。

## よくある操作例

### 例1: 新しいプロジェクトを組織に追加する

1. 新しいJiraプロジェクトがシステムに同期される（または管理者が追加する）
2. 「未分類プロジェクト」パネルに表示される
3. 割当アイコンをクリックして対象の組織を選択して紐付ける
4. ダッシュボードの組織別ヒートマップに反映される

### 例2: 組織改編でプロジェクトを移管する

1. 「未分類プロジェクト」パネルには表示されない（既に割当済み）
2. **現時点ではUIから直接変更できないため、管理者にデータベース上での変更を依頼してください**

## 注意事項

- 組織が1件も登録されていない場合、割当アイコンはグレーアウトされて操作できません。先に組織を作成してください
- 1つのプロジェクトは1つの組織にのみ属することができます
