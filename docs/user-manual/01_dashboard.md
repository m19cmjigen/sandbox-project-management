# ダッシュボード

ダッシュボードはログイン直後に表示されるトップページです。全社のプロジェクト・チケット状況をひと目で確認できます。

## 画面構成

```
┌─────────────────────────────────────────────────────────────────┐
│ ダッシュボード                                                   │
│                                                                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│  │総プロジェクト│ │遅延プロジェクト│ │注意プロジェクト│ │正常プロジェクト│          │
│  │    25    │ │    5     │ │    8     │ │   12     │          │
│  │チケット:500│ │チケット:50 │ │チケット:100│ │チケット:350│          │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘          │
│                                                                 │
│ 組織別プロジェクト状況                                           │
│ カードをクリックすると該当組織のプロジェクト一覧を表示します         │
│                                                                 │
│  ┌─────────┐  ┌──────┐ ┌──────┐ ┌──────┐                      │
│  │ 開発本部  │  │ 開発部 │ │ 営業部 │ │ 企画部 │                      │
│  │  RED    │  │GREEN │ │ RED  │ │YELLOW│                      │
│  └─────────┘  └──────┘ └──────┘ └──────┘                      │
└─────────────────────────────────────────────────────────────────┘
```

## サマリカード

画面上部の4枚のカードで全社状況を確認できます。

| カード | 表示内容 |
|---|---|
| 総プロジェクト | プロジェクト総数（サブテキスト: チケット総数） |
| 遅延プロジェクト | 遅延 (RED) プロジェクト数（サブテキスト: 遅延チケット数） |
| 注意プロジェクト | 注意 (YELLOW) プロジェクト数（サブテキスト: 注意チケット数） |
| 正常プロジェクト | 正常 (GREEN) プロジェクト数（サブテキスト: 正常チケット数） |

## 組織別ヒートマップ

画面下部では、組織ごとのプロジェクト遅延状況をカード形式で確認できます。

### カードの見方

各カードには以下の情報が表示されます。

- **組織名**: カード上部に表示
- **遅延ステータス**: カードの背景色で判断（赤=遅延、橙=注意、緑=正常）
- **プロジェクト数**: 配下のプロジェクト総数と各ステータスの内訳

### カードをクリック

組織カードをクリックすると、その組織に属するプロジェクトの一覧ページに移動します。

```
例: 「開発本部」カードをクリック
  → プロジェクト一覧ページ（開発本部でフィルタ済み）へ移動
```

### ヒートマップの階層構造

カードは本部（レベル0）を大きく左側に、その配下の組織を右側に小さく並べて表示します。

```
┌─────────┐  ┌──────┐ ┌──────┐
│  本部A   │  │ 部A  │ │ 部B  │  ← 本部Aの配下
└─────────┘  └──────┘ └──────┘
┌─────────┐  ┌──────┐
│  本部B   │  │ 部C  │            ← 本部Bの配下
└─────────┘  └──────┘
```

## ページの更新

ブラウザの再読み込み（F5 または Ctrl+R）でデータを最新状態に更新できます。
